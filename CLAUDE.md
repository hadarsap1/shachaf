# הנחיות לעבודה על הפרויקט

## תקשורת
- **לפנות למשתמש בלשון זכר, תמיד** (לא לשון נקבה ולא ניסוחים מעורבים כמו "תגידי/תגיד").
- לענות בעברית.

## פקודות
- `npm test` — בדיקות יחידה (vitest)
- `npm run test:rules` — בדיקות חוקי Firestore באמולטור
- `npm run lint` — ESLint (יש שגיאות/אזהרות קיימות בבסיס — לא להוסיף חדשות)
- `npm run build` — בניית פרודקשן (Vite)

## פריסה
- דחיפה ל-`main` פורסת אוטומטית: פרונטאנד ל-Vercel, וחוקי Firestore/Storage
  דרך GitHub Actions (`deploy-firebase-rules.yml`) כשקבצי החוקים משתנים.
- סגנון הריפו: קומיטים על ענף עבודה, מיזוג `--no-ff` ל-main עם הודעת "merge: ...".

## פרטיות — כללי ברזל
- כל שימוש חדש במידע אישי מחייב עדכון `src/lib/consent.js` (PURPOSES) והעלאת
  `CONSENT_VERSION` — המשתמשים יתבקשו לאשר מחדש.
- אין להציג פרטי משתמש או ילדיו לפני שהמשתמש אישר את גרסת התקנון הנוכחית
  (`hasConsented` / `childHasConsentedParent`).
- כל צ'ק-בוקס הסכמה חדש מתועד ב-`consentLog` דרך `logConsent`.
